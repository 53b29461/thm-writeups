<body>に<p>Username or password invalid</p>が追記されて返ってきた
この文言をログイン失敗時の目印としてヒドラを実施する
hydra -l admin -P /usr/share/wordlists/rockyou.txt 10.48.143.111 http-post-form "/admin/:user=^USER^&pass=^PASS^:F=Username or password invalid"

コマンドの構成要素のおさらい

  • -l admin: ユーザー名を admin に固定します。

  • -P /usr/share/wordlists/rockyou.txt: 試行するパスワードのリストを指定します。（※もし rockyou.txt が解凍されておらず .gz のままであれば、実行前に sudo gzip -d /usr/share/wordlists/rockyou.txt.gz で解凍してください）

  • 10.48.143.111: ターゲットのIPアドレスです。

  • http-post-form: POSTメソッドで送信されるWebフォームを攻撃するためのHydraのモジュールです。

  • "/admin/:user=^USER^&pass=^PASS^:F=Username or password invalid": " の中はコロン（:）で3つのブロックに分かれています。

    1. パス: /admin/

    2. リクエストボディ（パラメータ）: HTMLの name 属性に合わせて user=^USER^&pass=^PASS^ と記述します。実行時、^USER^ には admin が、^PASS^ には辞書の単語が次々と代入されて送信されます。

    3. 失敗の判定基準: F=Username or password invalid とすることで、レスポンスの中にこの文字列が含まれていれば「失敗」、含まれていなければ「成功」とHydraが自動判定します（F= は Failure の意味です）。
